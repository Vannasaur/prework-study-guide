HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The h1-h6 elements represent headers, there should only be one h1 per page
  • The ul, ol, and li elements represent unordered lists, ordered lists, and list items respectively
  • The br element creates a line of empty space, or line break, between two blocks of content
  • The img element contains info about images displayed
  • The img element must contain the src attribute which defines the location of the image via url
  • The alt attribute contains text that describes the appearance and functionality of the image
  • The header and footer elements appear at the top and bottom of the page respectively

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • CSS can be used in three ways: inline CSS, internal CSS style sheet, and external CSS style sheet
  • External style sheets must be linked to the HTML file in order to be applied
  • Code for linking external style sheet to HTML file: < link rel="relationship between docs" href="location (URL) of external source" >
  • CSS rule contains a selector and declarations
  • A selector defines the element or attributes to which the rules, or declarations, will apply
  • Declarations contain a CSS property and the value for that property, written as {property:value;}
  • Multiple elements or attributes can be styled in the same code as selectors separated by a comma

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: adds modifications in the current working branch to the staging area, the -A flag indicates that we want to add all changes
  • The staging area informs Git that the changes will be added to the next commit
  • git commit -m "message describing commit": pushes changes to commit with the message listed
  • git pull origin main: checks base branch for any changes and will let you know if it's up to date
  • git push origin branch-name: pushes changes we have locally to remote GitHub branch
  • A pull request, or PR, is used to merge a comparison branch into the base branch
  • Remember to frequently sync the local version of your code with the remote version in GitHub
